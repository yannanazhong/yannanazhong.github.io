一天造了一个hexo窝

Hello Human。我正式入驻我的杂草窝了，从此，我也算是有‘窝’一族了！成功入驻后，我也同大家分享下入驻过程的流程，虽然网上已经有很多很多很多关于’使用HEXO搭建博客及配置’的教程，但，但，但，蛋，蛋…

nodejs + git + hexo + github

1.下载git，安装

2.下载nodejs，安装
[–下载，安装即可–]

3.使用npm安装hexo

  1. 打开git，输入命令npm install hexo -g
  2. 新建一个文件夹（名称位置自定），打开文件夹右键，选择“Git Bash Here”，再次打开git后，输入命令
    npm init
  3. 初始化成功后，新建文件夹里面就包含了hexo所需文件
  4. 生成静态页面：
    npm g
  5. 本地启动测试：
    npm s
  6. 浏览器进 http://localhost:4000/ ，出现Hello World界面后，说明配置成功。

[–按提示 ctrl+c退出–]

4.部署本地文件到github

  1. 首先，你得有一个github账号，首先，你得有一个github账号，首先，你得有一个github账号
  2. 创建一个 New Repository (相当于一个仓库，作用：放代码)，Repository name名称格式必须为youname.github.io [–youname是指你自己的名字–]，必须必须必须的.github.io
  3. 重点重点来了：配置_config.yml [根目录下]

  4. 打开后在最后deploy处添加

    1.reopitory地址为你github新创建地址，地址后要加上 .git
    2.hexo的配置文件中任何‘ : ’后面都是带一个空格的
    3.如果配置以下命令出现ERROR Deployer not found : github 则执行命令 npm install hexo-deployer-git --save'
    4.重复如上几点最终可成功配置，记得：空格 地址

  1. 生成静态页面： npm g
  2. 推送到github： npm d
  3. 在浏览器访问 yourname.github.io ,然后就能看到自己的博客

这里有嫩草
解决hexo deploy时重复输入用户名密码的问题：
将_config.yml中repository地址修改为： git@github.com:yourname/yourname.github.io.git
